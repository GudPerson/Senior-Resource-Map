# CareAround SG User Guide Foundation

This document is the working base for a future CareAround SG user guide, live tutorial, help hints, and training material.

It is written for product planning first. It is not final user-facing copy yet.

The first public/standard-user guide draft now lives in `docs/user-guide.md`. Keep this foundation document as the broader planning base for future public guide updates, partner/admin operations guides, live tutorial hints, and training material.

## Guide Goals

The guide should help people answer four simple questions:

1. What can I do here?
2. What should I do next?
3. Why am I being asked for this information?
4. What should I check before I rely on or share the information?

The guide should avoid technical labels where possible. When a technical or operational term must appear, explain it in plain English.

## Audience Groups

| Audience | Main need | Guide tone |
| --- | --- | --- |
| Guest | Find and understand public resources quickly. | Simple, reassuring, low commitment. |
| Standard user | Save resources, create maps, personalise results, and share with family or helpers. | Practical and step-by-step. |
| Caregiver or family member | Prepare resource options for someone else. | Scenario-based and action-oriented. |
| Partner staff | Manage resources, private notes, memberships, translations, and safe exports. | Clear operating instructions with cautions. |
| Regional admin | Manage service areas, users, partners, resources, and data quality. | Structured, checklist-based. |
| Super admin | Maintain the full system safely. | Control-focused with regression and privacy reminders. |

## Core Terms To Teach

| Term | Guide explanation |
| --- | --- |
| Resource | A place, programme, service, promotion, or support item in CareAround SG. |
| Place | A real-world location with an address. |
| Offering | A programme, service, promotion, event, or support activity. |
| Saved Resource | A resource you kept for later. |
| My Directory | Your personal list of saved resources. |
| My Map | A map you build from saved resources. |
| Shared Map | A view-only map link that others can open. |
| Member link | A QR or link that connects your account to a place. |
| Partner-only | Information only authorised partner/admin accounts can see. |

Avoid teaching "asset", "hard asset", "soft asset", "CTA", or "boundary" to normal users.

## Suggested Guide Structure

### Part 1: For Everyone

1. **What CareAround SG Is**
   - A place to find senior-related resources in Singapore.
   - A way to save useful resources.
   - A way to make simple maps and directories for planning.
   - Not emergency, medical, financial, legal, or government-service advice.

2. **Choosing Language And Display Settings**
   - Pick English, Mandarin, Malay, or Tamil from the language selector.
   - Use contrast and text-size controls for readability.
   - Explain that some legal/admin content may remain English in V1.

3. **Finding Resources**
   - Open Discover.
   - Search by name, category, service, or postal code.
   - Use current location or a postal code to see what is nearby.
   - Switch between list and map views where available.

4. **Reading A Resource Detail Page**
   - Check the description, address, schedule, contact, and linked places.
   - Check whether the resource is a place or an offering.
   - Use directions only after confirming the resource is still suitable.
   - Verify programme availability, price, eligibility, and opening hours with the provider.

5. **Saving Resources**
   - Sign in to save resources.
   - Saved resources go into My Directory.
   - Saving does not register the user for a programme.

### Part 2: For Standard Users And Caregivers

6. **Using My Directory**
   - View saved resources.
   - Search and sort saved resources.
   - Remove resources that are no longer useful.
   - Understand "No longer available" and "Not shown on map".

7. **Creating My Maps**
   - Create a map from saved resources.
   - Add or remove resources.
   - Rename and describe the map.
   - Set a distance anchor such as home or another postal code.
   - Use the map for family discussion or care planning.

8. **Printing, Saving, And Sharing Maps**
   - Open print-friendly view.
   - Save as image.
   - Share a view-only map link.
   - Copy a shared map into your own My Maps when signed in.
   - Be careful before sharing personal planning information.

9. **Personalising Recommendations**
   - Add optional profile details.
   - Explain why postal code and eligibility details can help.
   - Explain that optional details can affect restricted offerings.
   - Keep wording gentle: these details help personalise, not judge.

10. **Linking Memberships**
   - Scan or open a membership link.
   - Sign in if needed.
   - Link the account to a place.
   - Understand that linked membership may help member-only offerings recognise access.

11. **Privacy, Cookies, And Terms**
   - Point users to `/privacy` and `/terms`.
   - Explain essential session cookies and browser preferences.
   - Explain that non-essential analytics/marketing cookies are not currently used.
   - Explain that privacy contact details must be confirmed before wider launch.

### Part 3: For Partners

12. **Partner Dashboard Basics**
   - Sign in through partner sign-in.
   - Understand the difference between public content and partner/admin tools.
   - Know which resources and users the partner is allowed to manage.

13. **Managing Places And Offerings**
   - Create or edit places.
   - Create or edit offerings.
   - Link offerings to places.
   - Set categories, tags, images, schedules, and visibility.
   - Keep public wording simple and useful.

14. **Using Partner-Only Notes And Files**
   - Add reference notes, pricing notes, service guides, or checklists.
   - Upload only authorised PDF/image files.
   - Understand who can view partner-only content.
   - Do not store unnecessary personal data or confidential information in partner-only content.

15. **Reviewing Translations**
   - English is the main source.
   - Mandarin, Malay, and Tamil can be auto prepared.
   - Check languages marked "Needs review" or "Missing translation".
   - Save review when wording is acceptable.
   - Staff-edited wording should not be overwritten silently.

16. **Importing Programme Material**
   - Upload flyers, calendars, or programme material.
   - Review extracted draft offerings before saving.
   - Check repeated sessions, full programmes, hidden status, categories, schedules, and linked places.
   - Imported content should be reviewed before it becomes public.

17. **Membership Links**
   - Generate or share QR/link flows for places.
   - Help users link their account to the correct place.
   - Explain what membership linking does and does not do.

18. **Safe User Export**
   - Export only when needed for legitimate operations.
   - Use filters or selected rows where possible.
   - Store downloaded files carefully.
   - Do not forward exported data outside authorised channels.

### Part 4: For Admins

19. **Service Areas And Audience Rules**
   - Explain subregions as management service areas.
   - Explain audience zones as selected postal-code areas.
   - Explain partner areas and restricted offerings.
   - Provide examples of who sees what.

20. **Visibility And Access Controls**
   - Public.
   - Hidden.
   - Scheduled hiding.
   - Member-only.
   - Partner-area restricted.
   - Audience-zone restricted.
   - Eligibility-based.

21. **Templates And Local Versions**
   - Explain reusable template offerings.
   - Explain child/local offerings.
   - Explain local overrides.
   - Provide examples of when to use standalone versus template.

22. **Workbook Import And Export**
   - Download the right workbook.
   - Fill only expected fields.
   - Use reports to fix row errors.
   - Avoid importing public data without review.

23. **Security And Data Protection Habits**
   - Use strong account practices.
   - Keep exports scoped.
   - Avoid uploading sensitive data into public fields.
   - Report suspected access issues.
   - Treat privacy contact replacement as a launch blocker.

## Live Tutorial And Hint Ideas

Use short hints inside the app only where they help the next action. Avoid large instructional blocks on every screen.

| Location | Hint idea |
| --- | --- |
| Discover empty state | "Search by place, service, or postal code to start." |
| First save action | "Saved resources appear in My Directory." |
| My Directory with saved resources but no maps | "Create a map when you want to group resources for planning or sharing." |
| My Map empty state | "Add saved resources from My Directory to build this map." |
| Share modal | "Anyone with the link can view this shared map. Share only with intended recipients." |
| Profile eligibility card | "These optional details help personalise recommendations and restricted offerings." |
| Translation review panel | "Start with languages that need attention. Ready translations can be left alone." |
| Partner-only editor | "Only authorised partner/admin accounts can see this content." |
| User export | "Export only the users you need, and handle the file carefully." |

## Training Prompts

These prompts can become live demo scripts or onboarding exercises.

### Public User Demo

1. Find resources near a postal code.
2. Open a resource detail page.
3. Save two useful resources.
4. Open My Directory.
5. Create a map from saved resources.
6. Print or share the map.

### Caregiver Demo

1. Search around a loved one's postal code.
2. Save a centre, a programme, and a service.
3. Create a map named after the care situation.
4. Set the distance anchor to home.
5. Export the map as an image for discussion.

### Partner Demo

1. Create or edit a place.
2. Add an offering.
3. Add partner-only notes and a reference file.
4. Review translations.
5. Confirm what a normal user can and cannot see.

### Admin Demo

1. Review users in a service area.
2. Check audience zones.
3. Import a small workbook or programme material.
4. Review warnings before saving.
5. Export a scoped user list and discuss safe handling.

## Copy And Language Rules For Guide Material

- Prefer "resource" over "asset".
- Prefer "place" over "hard asset".
- Prefer "offering", "programme", or "service" over "soft asset".
- Prefer "My Maps" over "Private Maps".
- Explain profile details as optional personalisation where possible.
- Avoid making users feel blocked or blamed.
- Use "restricted" instead of "locked" for access rules.
- Keep legal and privacy wording clear, but do not over-simplify legal meaning.
- For translated guide material, use reviewed human wording for official release.

## Launch Blockers And Open Decisions

These items should be resolved before a wider public launch guide is treated as final:

- Confirm public privacy/legal contact details.
- Decide whether legal pages stay English-only or receive reviewed translations.
- Decide which admin screens will remain English-only in V1.
- Confirm the safe-handling policy for user exports.
- Confirm who reviews Mandarin, Malay, and Tamil resource translations.
- Decide whether partner-only notes/files need content rules or prohibited-content examples.
- Refresh `docs/session-handoff.md`, which is currently historical.
